# 메모리 누수가 발생 하는 이유

// 메모리 leaks 없음.
int main()
{
	char *str = "0123456789";
	char *a;

	a = ft_substr(str, 6, 10);
	printf("%s\n", a);
	while (1)
		;
	return (0);
}

// 메모리 leaks 발생.
int main()
{
	char *str = "0123456789";

	printf("%s\n", ft_substr(str, 6, 10));
	while (1)
		;
	return (0);
}

  • 결과

    • ft_substr을 printf 에 직접 넣어줬을때 메모리 누수가 발생.
    • 따로 포인터로 만들고, printf 로 돌렸을땐 메모리 누수 없음.

  • 앞서 토론했던 메모리 누수 부분에 있어서의 문제점은 ft_substr에서 불필요한 공간까지 메모리를 할당해줘서 그 부분이 불필요한데 생겼기 때문에 누수라는 말이 있었다.

  • 하지만 위 예시의 실험으로는 그 결과와 차이 없이 메모리 누수가 없을때도 있었고 생길때도 있었다.

  • 이러한 차이를 알아보기 위해서 여러 링크를 통해서 자료를 찾아보았다.

  • 중점

    • 메모리 누수가 발생하는 주된 이유는 할당한 메모리의 주소값을 잃어버리는 경우가 대다수 이다!
    • 쉽게 말해서 char* a = (char *)malloc(5); 이렇게 할당한 메모리의 주소를 포인터 변수에 담아주면 주소값을 지닌 a 가 존재하기 때문에 free(a) 로 쉽게 메모리를 해제할 수 있지만, 변수에 담지 않고 사용하는 경우 (printf("%s\n", ft_substr(str, 6, 10));) 에는 사용 후 해제를 할 주소를 담고 있지 않아서 메모리 누수가 발생한다.

# (추가) free 함수는 어떻게 메모리 크기를 알고 해제 시켜줄까?

  • 질문
    : free 함수는 매개 변수로 크기를 허용하지 않습니다. free () 함수는 포인터 만 주어도 얼마나 많은 메모리를 해제 할 수 있는지 어떻게 알 수 있습니까?

  • 답변
    : 다음은 free ()가 할당 해제 할 메모리 크기를 알 수 있도록 메모리 크기를 저장하는 가장 일반적인 방법입니다. 메모리 할당이 완료되면 할당 된 실제 힙 공간은 요청 된 메모리보다 한 단어 더 큽니다. 추가 단어는 할당 크기를 저장하는 데 사용되며 나중에 free ()에서 사용됩니다.

  • 여기에서의 중점은 malloc 의 크기가 얼마이든 할당이 된다면, 주소값을 지닌 상황에서 free() 를 이용하여 메모리를 해제해줄 수 있다는 것이다.

# 7. 학습 내용에 대한 개인적인 총평

  • 계속해서 슬랙, 뮬리넷 등에서 이슈였던 메모리 누수에 대해서 심화적으로 생각하고, 검색하고, 이해해보았다.
  • 메모리 누수가 어떤식으로 발생되고 왜 발생되는지에 대해서 찾아보고 들었던 부분이 많으나, 이해가 되지 않는 부분이 계속 있었는데 이러한 부분들을 다시 이해하고 진도를 나갈 수 있어서 뜻깊은 시간이었다.
  • 개념적인 정리에 대한 인터넷 상의 글들에는 부족한 부분들이 많아, 양질의 학습을 위해서는 정확한 공부를 위한 웹사이트를 알고 있거나 책을 통해 학습해야 될 것같다.
  • 정확한 내용으로 공부를 할 수 있는 환경에 대해서 생각해볼 수 있는 시간이었다.
  • 많이 찾으면 결국엔 정확한 내용을 찾겠지만, 여러가지 필터링되지 않은 정보들이 오히려 공부에 혼란을 야기하기도 하고, 시간적인 손해를 많이 보게 되는 경우가 많다. 여러 사람들에게 제대로 공부할 수 있는 정보를 물어봐야겠다.
