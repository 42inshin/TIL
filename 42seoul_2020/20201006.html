# 2020-10-06(화)

# 1. 학습 날짜

  • 2020-10-06(화)

# 2. 학습 시간

  • 18:30 ~ 10:30(대구 앞산 카페)

# 3. 학습 범위 및 주제

  • Libft 에 대한 기본적인 이해 및 함수 구현.
  • ft_putchar_fd
  • ft_putstr_fd
  • ft_putnbr_fd

# 4. 동료 학습 방법

  • 해당 사항 없음.

# 5. 학습 목표

  • Libft 과제에 대한 이해
  • 라이브러리 안 각각의 함수에 대한 이해와 구현

# 6. 상세 학습 내용

  • 과제를 시작하기 앞서 12페이지의 LIBFT 문서에 대해서 읽어 보았다. 필요한 함수가 Libc 함수에서만 24개이며, 추가 함수에서 10개를 더 만들어야했다. 내용 상, 추가 함수를 이용해서 libc 함수를 만드는 경우도 있다고 적혀있어서, 추가 함수에서 쉬운 것들부터 하기로 계획했다. 기존에 알고 있던 ft_putchar, ft_putstr, ft_putnbr에 fd(파일디스크립터)를 추가한 것부터 하기로 했다. 
<file descriptor(파일 디스크립터)>
- 유닉스 시스템의 모든 것은 파일이리고 한다.
- 유닉스 시스템에서 프로세스가 각 파일들에 접근할 때 파일 디스크립터라는 개념이 필요하다.
- 프로그램이 프로세스로 메모리에서 실행을 시작할때 기본적으로 파일 디스크립터가 할당된다.
- 표준 입력(0), 표준 출력(1), 표준 에러(2)라는 정수가 할당된다.
- 예로, write(1, &c, 1); 에서 맨처음 위치가 file descriptor 이다.

# 7. 학습 내용에 대한 개인적인 총평

  • Bad : 생각한 만큼에 비해 공부 효율성이 떨어진다. 생각보다 모르는 부분이 많아서 찾아보는 시간이 매우 길다. 효과적으로 공부하는 방법에 대해 생각해봐야 할 것 같다.
  • Good : 일단 첫 시작을 가볍게 하면서 문제에 친숙해지려 했던 부분에 칭찬해주고 싶다. 차츰차츰 과제를 진행하여 끝을 향해 도약하는 멋진 나를 기대해 보자.
