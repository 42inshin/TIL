# Get_next_line 선행지식

  1. file Descriptor
  • fd = open("text.txt", O_RDONLY);
  • 유닉스 관련 컴퓨터 운영체제가 만든 파일 또는 소켓의 지칭을 편하게 하기 위해 부여된 음이 아는 정수(c언어에서 int 표시).
    • 0: 표준입력 (Standard Input)
    • 1: 표준출력 (Standard Output)
    • 2: 표준오류 (Standard Error)
  • 0, 1, 2는 할당되어 있어서 3번 부터 파일 디스크립터를 부여함.
  1. read()
size_t read(int fd, void *buf, size_t bytes)
  • fd를 bytes 만큼 읽어 buf에 저장한다.
  • 반환 값
    • 읽기 성공 시, 읽은 바이트 수
    • 읽기 실패 시, -1
    • 읽기 완료 시, 0
  1. static 변수
  • 프로그램이 종료되었을 때도 데이터 영역에 값이 남아있을 수 있도록 변수 앞에 선언해주는 명령어.
  1. gcc -D
  • -D BUFFER_SIZE=xx.
  • #define 을 통해 BUFFER_SIZE 를 정의해주어야 함.

# 목표

  1. GNL 함수를 사용하면 해당 fd의 텍스트를 EOF 가 나올 때 까지 한번에 한 줄씩 읽는다.
  2. 버퍼사이즈가 커서 파일을 끝까지 읽더라도, 다음 호출 시 두번째 line 부터 시작해야한다.
  3. file 또는 redirection, 표준입력(stdin)으로부터 읽었을 때 정상작동해야한다.
  4. BUFFER_SIZE 가 1, 9999, 1000만 일때도 정상작동해야한다.

# 해결과정

  1. read함수에 사용할 buf를 만든다.
  2. read한 버퍼를 백업할 static backup 변수를 만든다.
  3. read(fd, buf, BUFFER_SIZE);
  4. buf를 static backup 에 담는다.
  5. backup에 \n 이 있는지 검사한다.
  6. 개행 유무
  • 개행이 있는 경우, 7번으로
  • 개행이 없는 경우 3번으로 돌아가 다시 읽으면서 backup 에 합쳐준다.(strjoin 함수)
  1. 개행이 있는 경우, backup을 \n 앞 뒤로 잘라서 앞은 **line 에 넣어주고, 뒷 부분은 다음에 쓰기 위해서 backup에 백업한다.

# 7. 학습 내용에 대한 개인적인 총평

  • GNL이 쉬울것이라 막연히 생각했는데, 생각보다 개념잡기와 해결 방법과 그에 따른 문제사항들에 대해서 생각해야 될 것들이 많았다. 이전부터 계속 헷갈리던 이중포인터 개념이 다시 나오기도하며, 라인 하나 출력하고 다음라인을 출력하기 위해 데이터를 담고 있는 방법이 이렇게 어려운 일인지 다시금 깨닫게 된다.
  • 영덕님(yuchoi)과 함께 슬랙콜을 이용하여 화면공유를 통해 GNL 설명을 들으면 문제점들을 찾아봤지만, 아직까지 만드는 구조에 대해서 완벽하게 인지하고 있지 않아서 피드백하기가 쉽지 않았다. 조금더 자세하게 코드를 구상하는 방법에 대해서 구상해야될 것같다. 아이디어를 구체적으로 적어서 그 아이디어를 바탕으로 코드를 짜는 것을 자주 해보도도록 해야될 것 같다.
