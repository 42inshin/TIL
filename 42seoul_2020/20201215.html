# 6. 상세 학습 내용

실제 코딩 시간 : 총 1시간.

# 메모리 구조

  • 머신 코드(machine code)
    : 프로그램이 실행될 때 그 프로그램이 컴파일된 바이너리가 저장된다.
  • 글로벌(globals)
    : 프로그램 안에서 저장된 전역 변수가 저장된다.
  • 힙(heap)
    : malloc 으로 할당된 메모리의 데이터가 저장된다.
  • 스택(stack)
    : 프로그램 내의 함수와 관련된 것들이 저장된다.

# 메모리 특징

  • 힙 영역에서는 malloc 에 의해 메모리가 더 할당될수록, 점점 사용하는 메모리의 범위가 아래로 늘어납니다.
  • 마찬가지로 스택 영역에서도 함수가 더 많이 호출 될수록 사용하는 메모리의 범위가 점점 위로 늘어납니다.
  • 이렇게 점점 늘어나다 보면 제한된 메모리 용량 하에서는 기존의 값을 침범하는 상황도 발생할 것입니다.
  • 이를 힙 오버플로우 또는 스택 오버플로우라고 일컫습니다.

# 7. 학습 내용에 대한 개인적인 총평

  • cs50 강의를 보면서 포인터의 개념에 대해서 더 자세히 익힐 수 있었고, 메모리의 구조와 동작법에 대해서 막연히 이렇구나 생각했던 부분에 대해서 더 상세히 이해할 수 있었다.
  • fopen, fread, pointer, heap, stack 에 대해 학습하고, 이러한 강의가 피신 전에 이루어졌다면 c 에 대한 학습을 진행할 때 더 큰 도움이 되었지 않았을까 생각해보며, 지금까지 모르는 분야의 개발 방향에 대한 학습을 꾸준히 하자고 생각을 하게된 계기가 되었지 않았을까 싶다.
  • 좀더 글들이 정리가 되었으면 좋겠는데, 계속 글을 쉽게 풀어쓰는게 익숙해져서 정확성보다는 감정을 호소하는 듯하다. 이러한 부분들을 적당히 바꿔서 좀더 개발자스러운 글들을 써보도록 연습해봐야겠다.
