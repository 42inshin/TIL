# 2020-12-14(월)

# 1. 학습 날짜

  • 2020-12-14(월)

# 2. 학습 시간

  • 14:00 ~ 18:00 (집)
  • 19:00 ~ 21:00 (집)

# 3. 학습 범위 및 주제

  • GNL 코드리뷰
  • 해커톤 준비를 위한 아이디어 회의

# 4. 동료 학습 방법

  • yuchoi 와 GNL 코드리뷰
  • yuchoi, kyukim 과 함께 디스코드를 통한 회의.

# 5. 학습 목표

  • 해커톤을 대비하여 아이디어를 모으고, 진행방향에 대해서 토의하자.

# 6. 상세 학습 내용

실제 코딩 시간 : 총 1시간.

# 내용

  • 42API, 라즈베리파이 및 아이패드를 활용하여 42서울 교육장에서 이용할 수 있는 서비스 개발

# 진행순서

  • 온라인 해커톤 진행(Zoom 활용 예정)
    • 16일: 아이디어 구성 및 개발환경 구축
    • 17일: 개발진행
    • 18일: 개발 완료 및 발표(온라인)

# 해커톤 아이디어

  1. 블랙홀 or 같은 과제 하고 있는 사람들 모아보기 😃
    • 5번 아이디어와 통합 가능 할 듯
  2. 42서울 and 기수별 자신의 진척도 비율(%)
  3. 얼굴 인식으로 프로필 및 정보 확인
  4. 인트라 아이디를 입력 받아 자리 위치 안내
    • 프린터 연결 가능하면 출력까지
  5. 로그인을 하면 과제에 따라 관련 슬랙 링크 보여주기

# 개발 목표

  • (공통) API 로 자료 가져오기
  • (공통) 라즈베리파이에 저장

# 알아두면 좋을 개념들

  • CRUD
  • uri
  • body
  • request
  • response
  • status code 200, 404, ~~~ 주요 몇 개
  • API 설명 페이지에서 사용할 만한 기능(함수) 및 반환값 읽어두기

# 7. 학습 내용에 대한 개인적인 총평

  • 4시간 정도의 시간으로 16~18일 간에 있을 해커톤에 대해서 팀원들가 이야기를 나누었다. 라즈베리파이, API 등 전혀 무지한채로 해커톤을 한다고 호기롭게 하다보니, 도무지 아이디어가 떠오르지 않아서 우리가 어떤것을 해야할지 생각하는데 시간을 많이 소요했다. 이번 해커톤에서 주요 핵심은 42API와 라즈베리파이를 이용하는 것인데 그것을 이용하여 만들어야 되는 아이디어는 웹기반의 아이디어와는 다른 문제여서 도무지 생각이 나지 않았다. 그래도 미리 이렇게 아이디어를 짜내다보니 처음에 있을 당혹감을 미리 준비할 수 있어서 해커톤 진행 때에는 목표를 향해 집중해서 나아갈 수 있지 않을까 생각해본다.
  • GNL 코드 리뷰를 진행하였는데, 나와 다르게 짠 코드를 읽으면서 과정을 다시 생각해볼 수 있는 것은 42서울의 큰 장점이라고 생각이든다. 같은 과제로 여러 코드를 보면서 더 좋은 코드가 어떤 것인지 생각해 볼 수 있는 것은 학습적으로 매우 도움이 된다.
