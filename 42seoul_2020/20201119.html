# 6. 상세 학습 내용

실제 코딩 시간 : 총 1시간.

  • cs50 강의

    • 새벽에 잠을 못이뤄 cs50 강의를 보면서 공부를 진행하였다.
    • 지금까지 피씬, 본과정에서 진행했던 부분을 통해서 배울 수 있었던 내용들이 많이 수록되어 있었다. 특히, 연결리스트는 이 강의를 들으면서 이해를 했다고 볼 수 있다.
    • 오버플로우, 중첩 루프, 자료형, printf의 형식 지정자, 연산자 등 기초적으로 배울 수 있는 부분들을 다시 한번더 보면서 디테일한 부분들에 대해서 더 알 수 있는 시간이었다.
    • gcc, clang 등의 컴파일러로 컴파일 시, -l 라이브러리 이름 옵션을 쓸 경우 라이브러리를 검색한다는 것도 강의를 통해 알게 되었다.
    • 전처리, 컴파일링, 어셈블링, 링킹 이라는 4단계 컴파일링에 대해서 배울 수 있었다.
      • 전처리 : 전처리기에 의해 수행된다. 예로 #include 와 같은 줄이 헤더 파일의 내용으로 포함된다.
      • 컴파일 : 전처리 이후 과정, C코드를 어셈블리어라는 저수준 프로그래밍 언어로 컴파일한다.
      • 어셈블 : 어셈블리 코드를 오브젝트 코드로 변환 시킨다. 0과 1로 이루어진 코드로 바꿔주는 작업. 소스 코드에서 오브젝트 코드로 컴파일 되어야 할 파일이 딱 한개라면, 컴파일 작업은 여기서 끝나나, 그렇지 않은 경우 링크라 불리는 단계가 추가된다.
      • 링크 : 만약 프로그램이 여러 개의 파일로 이루어져 있어 하나의 오브젝트 파일로 합쳐져야 한다면 링크라는 컴파일 마지막 단계가 필요하다. 여러 개의 오브젝트 코드 파일을 실행 가능한 하나의 오브젝트 코드 파일로 합쳐준다.

  • junhykim : ft_split 에 관한 설명

    • ft_split 에 대해 여러번 설명을 했으나, 계속 이해가 가지 않는 부분이 있는듯 하여 junhykim 에게 다시금 설명을 해주게 되었다.
    • 스플릿 함수에서 첫번째로 중요한 것은 이중포인터(tab)에 각각 스플릿 된 글자의 주소값을 넣어준다는 것이다.
    • 그렇기 때문에 나누어질 문자들의 갯수를 파악하는 것이 중요하다.
    • 스플릿 하는 c 문자가 연속으로 있거나 맨앞 맨뒤에 있는 경우, 카운트를 해주지 않아야한다.
    • 그 뒤로 나눠지는 문자들을 각각의 크기마다 메모리 할당하여 이중포인터(tab[j])와 이어주는 것이 중요하다.
    • 예외처리로, tab 이 malloc 할당 되지 않은 경우와 각각의 문자들을 malloc 할 때 메모리 할당 되지 않는 경우 전부 free 해주는 것이다.

# 7. 학습 내용에 대한 개인적인 총평

  • 레포트 쓰다보면 그날의 기억이 잘 떠오르지 않을 때가 많다. 하루종일 컴퓨터 앞에서 앉아 있었지만 오늘 뭔가 한게 없다고 느껴질때가 종종 많아지곤한다. 아무래도 계획적으로 더 움직이지 못하고 알차게 쓰지 못한 흘러가는 시간이 많아서 그렇게 느껴질 때도 있다. 아니면 나의 공부가 아닌 남을 도와줄때, 정작 어떤걸 도와줬는지 생각나지 않을 때가 많아진다. 낮밤이 바뀌고 새벽마다 하고 아침에 자는 습관이 기억을 점점 더 못하게 만드는게 아닐지도 모르겠다. 이제는 다시 정상적인 시간으로 되돌리고 시간을 온전히 써야될 것 같다. libft 를 열어놓은지 어느덧 2달이 되었다는게 충격적이다. 계속해서 뭔가를 하긴 했지만, 확실하게 말할 수 있는건 게을렀고 여전히 프로그래밍에 모르는 것들이 많다. 꾸준하게 계속 앉아서 코딩을 하는 버릇을 들여야겠다.
