# 네트워크

# IP 주소

  • 인터넷 프로토콜(IP) 은 인터넷의 정보들이 어떻게 전송되는지를 정의하기 위한 규칙들.
  • 인터넷 프로토콜 중 일부는 인터넷 상에 있는 각 장치에 대해 IP 주소를 할당해주는 것과 관련이 있다.
  • IP(인터넷 플토콜)은 여러 버전을 거쳐왔다.
  • 가장 최근 버전은 IPv6, 기존의 IPv4 프로토콜을 대체하기 위한 새로운 버전.

# IPv4 주소, IPv6 주소

  • IPv4 : 4개의 10진수로 표현(0 ~ 255)
  • 32bit = 8bit * 4 (약 43억개)
  • 이것이 부족해서 만든것이 IPv6 표준
  • IPv6 : 8개의 16진수로 구성(1개 0000 ~ffff)
  • 128bit = 16bit * 8

# 사설 IP 주소

- 10.#.#.#
- 172.16.#.# ~ 172.31.##
- 192.168.#.#

# localhost

  • IPv4 : 127.0.0.1 (자신의 컴퓨터 를 의미)

  • IPv6 : ::1 (0:0:0:0:0:0:0:1 의 약자)

  • 로컬 컴퓨터를 원격 컴퓨터인 것처럼 통신할 수 있어 테스트 목적으로 주로 사용.

  • 무선 인터넷을 사용할 때 연결되는 순서
    : Labtop - AP(엑세스 포인트) - 스위치 - 라우터 - 인터넷
    ex) AP의 예시 : 무선 공유기

# DHCP(동적 호스트 구성 프로토콜) : 컴퓨터 IP주소를 할당 (자동)

# DNS(도메인 이름 시스템) : URL을 받아서 IP 주소로 변환

# <그 외의 프로토콜>

  • TCP (전송 제어 프로토콜) : 인터넷으로 보내진 모든 데이터 패킷이 전송되도록 보장, 인터넷을 통해 전송되는 정보 패킷이 어떤 서비스를 위한 것인지 (웹 검색, 이메일 등) 알 수 있도록 보장.
  • HTTP(하이퍼텍스트 전송 프로토콜) : 웹 브라우저들과 서비들이 통신할 수 있도록 도와주는 프로토콜.
  • 패킷 : 인터넷에서 전송되는 데이터 단위

# 라우터 : 인터넷상의 메시지(패킷)를 원하는 경로로 전달하는데 도와주는 장치

  • 라우팅 테이블 : 각 데이터 패킷이 목적지 IP 주소에 따라 어디로 보내져야 할지 알 수 있는 정보
  • 라우팅 : 네트워크 안에서 통신 데이트를 보낼 경로를 선택하여 보내는 과정.
  • 정보는 한번에 하나씩 보내지는데 그 작업이 꼬이는 경우, 교착 상태에 빠질 수 있다

    교착 상태 : 두 개 이상의 작업이 서로 상대방의 작업이 끝나기만을 기다리고 있기 때문에 결과적으로 아무것도 완료되지 못하는 상태.

# TCP : transmission control protocol 전송 제어 프로토콜

  • 데이터를 순서 있는 패킷들로 분해
  • TCP는 각 패킷에 대해 순서에 맞게 번호를 매김
  • 패킷들을 올바른 순서로 재조립
  • 데이터에 포트 번호를 할당
    • 21 FTP
    • 25 SMTP
    • 53 DNS
    • 80 HTTP
    • 443 HTTPS

# Tip 로컬 주소에서 다른 도메인 주소까지 통과하는 라우터를 보는 명령어

>$ traceroute -q 1 www.naver.com

# 7. 학습 내용에 대한 개인적인 총평

  • 함께 공부하는 준현님의 libft 를 계속 도옴을 주어 libft 과정을 거의 끝낼 수 있었다. 원래의 계획은 이번주에 libft 평가를 보고 netwhat 까지 마무리할 계획이었으나, 파일 업로드 실수로 다시 하루를 기다려야 되는 상황이다. 그러나 평가 포인트를 잃지 않고, 실수를 바로잡을 수 있어서 다행이다.
  • 계획이 변경되어 네트워크에 대한 공부를 주로 학습하였다. 네트워크 관련해서는 거의 대다수의 것들이 모르는 내용이 많았으나 정리가 잘되어있는 강의를 찾아서 그걸로 공부를 할 수 있었다. 이 부분의 강의를 다 듣고, 다시 한번더 정리를 하면서 netwhat 을 바로 넘어갈 수 있도록 준비해야 되겠다.
  • 앞으로 얼마남지 않은 블랙홀을 늘리고, 블랙홀 기간에 연연하지 않고 자기주도적 학습을 진행하도록 하루의 계획을 성실하게 수행하도록 하자.
