# 1. c 프로그램 빌드 절차

  • 컴파일 : (*.c) 파일을 컴파일하여 Object 파일(*.o)를 생성함.
  • 링크 : 링커를 실행해서 main.o 를 포함한 Object파일을 통해 a.out인 실행 파일을 생성한다.

# 2. Makefile 구조

  • Makefile은 3가지로 구성됨.

  • 대상(Target), 의존관계(Dependency), 명령(Recipe)

    <target> : <dependency>
  (tab)<Recipe>
    • 대상 : 빌드 대상 이름. 명령에 의해 최종적으로 생성되는 파일명, (오브젝트 파일이나 실행 파일)
    • 의존 관계 : 빌드 대상이 의존하는 Target이나 파일 목록. 여기에 나열된 대상들을 먼저 만들고 빌드 대상을 생성.
    • 명령 : 빌드 대상을 생성하는 명령. 여러 줄로 작성할 수 있으며, 각 줄 시작에 반드시 Tab 문자로 된 indent 가 있어야함.
    // 예시
a.o : a.h a.c
  	gcc -c a.c

a.out : a.o b.o main.o
  	gcc -o a.out a.o b.o main.o

# 3. Makefile 명령어.

  • 실행 : make
  • 삭제 : make clean
    clean:
 	rm -f $(OBJS) $(OBJS_B) // 오브젝트파일, 보너스 오브젝트 파일 제거
  • 선별적 실행 : make <타겟이름>

# 4. 변수

  • Makefile 내에서 변수를 정의할 수 있다. (대문자로 지정)
    • CC = gcc : 변수 지정
    • $(CC) 로 사용할 수 있음.
    CC = gcc

 a.out : a.o b.o main.o
 	gcc -o a.out a.o b.o main.o

 a.out : a.o b.o main.o
 	$(CC) -o a.out a.o b.o main.o

 // 위에 두 구문은 같은 내용임.
    • 그 밖의 변수
      • CFLAGS : c 컴파일에 제공할 추가 플래그. (-Wall -Wextra -Werror)

# 5. PHONY

  • .PHONY 에 Target을 등록해두면 target과 같은 이름의 파일이 존재하더라도 make 의 명령어로 실행을 합니다.
  • 반대로 PHONY 에 등록하지 않고, clean 이라는 파일이 디렉토리에 존재할 경우, make clean 명령을 사용하면 make: 'clean' is up to date. 와 같은 명령어가 나오면서 기존의 명령을 무시하게 될 수 있습니다.
  • 이러한 경우를 막기위해서, 만들어 놓은 target 들을 PHONY 에 등록하는게 좋습니다.
      .PHONY: all bonus clean fclean re .c.o

# 6. 패턴 사용하기(자동 변수)

%.o : %.c %.h
  $(CC) $(CXXFLAGS) -c $<
  • %.o 의 % 는 *.o 와 같다고 볼수 있음.
  • $< 는 dependency의 첫번째 파일 이름.
  • 기타 자동변수
    • $@ : 타겟 이름
    • $< : 의존 파일 목록 첫번째 파일.
    • $^ : 의존 파일 목록 전체
    • $? : 타겟 보다 최신인 의존 파일
    • $+ : $^와 비슷하나, 중복된 파일 이름들 까지 모두 포함.
