# libft 평가

  • libft 수정을 마치고 드디어 평가를 받을 수 있었다. 완벽을 기하기 위해 시간을 많이 쓴 만큼 평가에 있어서 어려운 부분은 어려운 부분을 없었다. 주로 substr, split의 코드 리뷰, Makefile 명령어가 잘 되는지에 대해서 확인을 하였으며 전부 이상없음을 함께 확인하였다.
  • taekang : 테스터기 확인 및 메모리 누수 관한 substr, split에 대해 중점적으로 확인해 주셨습니다.
  • hoslee : 메모리 누수에 관한 오류들 확인해보고 코드에 대해서 좋게 말씀해주셨습니다.
  • mchae : shift + z z 누를시 vim 에서 저장 후 나가기 팁 알려주심.

# netwhat 시험

  • 사전에 netwhat 문제를 연속 3번 풀어보면서 내용을 익히고, 각각의 내용들에 대해서 공부하면서 정리를 나름 했다고 생각하여 시험을 치게 되었다.
  • 하지만, 시험을 그냥 머리로 계산해서 빠르게 풀 수 있다는 오만한 생각으로 10분이라는 시간제한 안에 풀지 못했다.
  • 계산기를 사용하여 10진법을 2진법으로 바꾸거나 호스트 주소 최대 갯수를 구하는 것은 도움을 받아야 될것 같다.

# printf 코드 수정

  • junselee 님과 잠깐 대화 도중, 오류를 수정해야 되는데 모르겠다는 부분이 있어서 같이 해보자고 얘기를 건내었다.

  • 첫번째, Makefile 에서 #include "libft.h" 의 경로를 바꿨을 때, 인식을 못하는 부분에 대해서 아무리 수정을 해도 문제가 있다고 하여 확인하였다.

  • Makefile 에서 header 파일의 경로를 설정하는 것은 gcc 옵션중 -I<경로> 인데, 코드를 확인해보니 중복으로 설정되어서 꼬여있었다. 혼자서 볼때는 그부분이 잘 안보였는데 다행히 같이 확인하면서 경로가 잘못되었던 것을 바로잡을 수 있었다.

      // 헤더파일 경로 ./src
  gcc -I./src

# 7. 학습 내용에 대한 개인적인 총평

  • 당일에 평가 포인트를 두배로 벌거나, 평가 받을 경우 포인트가 깎이지 않는 평가 이벤트가 열렸다.
  • libft 를 드디어 성공해서 20일 이하로 떨어질 뻔한 위기상황에서 83일로 블랙홀에서 멀어질 수 있었다. 물론 지금도 진도가 많이 느린편이지만, 지금까지 하면서 알게된 지식들을 기반으로 앞으로의 과제들을 담담하게 넘어갈 수 있다면 지금의 시간들이 보람있었다고 생각이 들지 않을까 싶다.
  • 어느덧 올해의 마지막을 향해가고 있다. 42서울을 알게되고 시작하면서 내가 코딩을 잘할 수 있을까. 이것으로 정말 미래를 보장받을 수 있을까 하는 고민이 많이 들었었다. 여전히 답을 찾지는 못했지만 답을 향해 꾸준히 나아가다보면 그 발자취에서 답을 얻을 수 있지 않을까 생각해본다. 여기서 멈추거나 쓰러지면 결국 후회를 할 것이고, 앞으로 나아가다보면 지금을 대견하게 여기지 않을까? 여러모로 공부에서 진척인 상황이 정신적으로도 힘겹게하는 요즘이다.
  • 얼른 코로나가 끝나고 제대로 다같이 모여서 진짜 동료학습다운 동료학습을 하는 날이 오길 기대해본다.
