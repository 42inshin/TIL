# 6. 상세 학습 내용

  • Ubuntu에서 code <file_name> 명령어를 적으면 vscode에서 편집할 수 있다.
  • norminette 실행 시 아래와 같은 오류 문구가 뜬다. Warning: the running version of Bundler (2.1.2) is older than the version that created the lockfile (2.1.4). We suggest you to upgrade to the version that created the lockfile by running `gem install bundler:2.1.4`.
    • 이 문제는 bundler 2.1.2 가 기본값으로 되어 있어서 뜨는 문구이다.
    • 해결을 위해서 uninstall을 하고 새로 설치하였으나, 전혀 변화가 없다.
    • bundler 문제해결2 링크에서 아래의 코드를 찾아 2.1.4 버전이 기본값이 되도록 수정하였다.
    • sudo gem install bundler -v 2.1.4 --default
    • 이제 문제의 문구가 뜨지않고 해결 되었다.
  • vscode 내에 42-comment Extenstions을 설치하였으나, windows에서 단축키가 먹히지 않음.
    • 이 문제를 해결하기 위해서 바로가기 키에서 키설정을 바꿔주었으나, 오류로 키가 바뀌지않음.
    • 찾은 방법은 ctrl + shift + p 에서 shortcut 검색하여 Preferences: Open Keyboard shorcuts (JSON) 클릭 후 다음을 추가함.
      // 42 comment
{ "key": "ctrl+shift+oem_2", "command": "extension.42-comment",
  "when": "editorTextFocus" }
    • ctrl+shift+/ 로 단축키 설정 완료
  • libft test
    • 본과정이 들어서면서 나오는 과제들은 라피신과는 다르게 방대한 내용(*.c, *.h, Makefile)을 담아야한다. 그렇기에 시간 소요도 크며, 기본적으로 쌓아햐 하는 지식 수준이 다르다. 특히 각각의 파일들이 제대로 구현 되었는지 테스트를 해보려면 엄청난 노동이 필요할 것이다.
    • 이러한 노동을 줄여줄 수 있는 해외 42코더들이 만들어 놓은 test들이 많다. 그중에 libft는 첫시작이라서 test할 수 있는 프로그램이 많다.
    • 하지만, 설치 후 기본적으로 필요한 파일들을 구성해야지만 제대로 동작하므로, 사전에 필요한 Makefile, libft.h, ft_*.c 파일들을 구성하는 법을 알아야 한다.
