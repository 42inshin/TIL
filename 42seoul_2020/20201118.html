# Makefile

  • Makefile 참고 링크

http://doc.kldp.org/KoreanDoc/html/GNU-Make/GNU-Make-4.html https://uareuni.tistory.com/47

		$(CC) $(INCS) $(CFLAGS) -c $*.c
		$(CC) $(INCS) $(CFLAGS) -c $<
		$(CC) $(INCS) $(CFLAGS) -c $?
		$(CC) $(INCS) $(CFLAGS) -c $< -o $(<:.c=.o)
		$(CC) $(INCS) $(CFLAGS) -c $< -o $(<:c=o)

  • 위 코드는 모두 같은 작업을 수행한다.

  • $<
    : $< 는 현재의 목표(Target) 파일보다 더 최근에 갱신된 파일 이름이다. .o 파일보다 더 최근에 갱신된 .c 파일이 있다면 자동적으로 컴파일이 된다. 가령 main.o를 만들고 난 다음에 main.c를 갱신하게 되면 main.c는 $<의 작용에 의해 새롭게 컴파일이 된다.

		INCS	= -I. #include src 의 경로 추가.
		LIBC	= ar -rcs #replace Creat sort
  • INCS : #include "libft.h" 의 경로를 추가하는 변수
  • LIBC : 정적라이브러리를 만들어 주는 명령어를 담은 변수
    • ar : archive 를 만들어주는 명령어
    • -r : replace
    • -c : create
    • -s : sort

# gcc option

  • -o : 만들어지는 파일의 이름을 지정해주는 옵션
  • -c : 목적파일까지 컴파일 단계를 수행하여 .o 파일을 만들어준다. 기본적으로 파일과 같은 이름의 .o파일을 만든다.
  • -I : -I 옵션으로 지정한 모든 디렉토리는 #include "file" 의 경우에만 검색됩니다. #include <file>에 대해서는 검색되지 않습니다.

# 7. 학습 내용에 대한 개인적인 총평

  • 학습을 진행하면서 스스로 알고 있다고 생각했던 부분을 막상 가르쳐주려고 하니, 제대로 정확히 알고 있다고 하기 어렵게 버벅거리는 모습을 보였다. 동료학습의 필요성의 예시로 볼 수 있다. 이처럼 동료와 함께 학습을 진행하고 서로가 서로에게 도와주면서 우리는 학습하면서 배운 내용을 확신으로 만들어 준다. 이러한 과정을 거치면서 계속된 학습을 한다면, 실력을 급격히 성장시킬 수 있지 않을까. 지금까지 어렵게만 느껴졌던, Makefile 만드는 과정과 libft의 진행순서들을 다시 복기하면서 오랜시간을 끌었던 과제를 무사히 잘 끝마칠 수 있을 것만 같은 기분이 든다.
