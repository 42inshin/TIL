# branch 가지 치기

  • branch 는 안전하게 격리된 상태에서 무언가 만들때 사용한다.
  • 여러 명이서 동시에 작업을 할 때, 다른 사람의 작업에 영향을 주거나 받지 않도록 메인 브랜치에서 자신의 작업 전용 브랜치를 만듭니다. 그리고 각자 작업을 진행한 후, 작업이 끝난 사람은 메인 브랜치에 자신의 브랜치이 변경 사항을 적용합니다. 이런 방식으로 독립적으로 특정 작업을 브랜치에서 수행하고 기록을 중간 중간 남기므로 문제 발생시 원인이 되는 작업을 찾아서 대책을 세우기 쉽습니다.
  • 평소에는 master 가지가 만들어있다.
  • git branch <브랜치이름> : 새로운 branch 생성
  • git checkout -b <브랜치이름> : 새로운 branch 만들고 갈아탐.
  • git checkout master : master 가지로 이동.
  • git branch -d <브랜치이름> : 만든 가지를 삭제.
  • git push origin <브랜치이름> : 원격 저장소에 저장.
  • git merge master <브랜치이름> : 수정사항을 master 에 추가

# 갱신 및 병합.

  • git pull : 원격 저장소의 변경내용이 로컬 디렉토리에 받아지고 병합됨.
  • git fetch : 받아짐. 충돌일어나지 않는 상태
  • git merge <브랜치이름>: 다른 가지에 있는 변경 내용을 현재 가지(주로 master)에 병합함.
  • git diff <원래 가지> <비교 가지> : 병합 전에 어떻게 내용이 바뀌었는지 비교. git add 후 사용가능.

# 해제, 초기화

  • git reset <파일>: git add 로 준비 영역에 있는 파일을 다시 해제함.

# 로컬 디렉토리에서 원격 저장소에 푸시하는 법

  1. 임의의 폴더 생성
  2. git init : 현재 디렉토리를 git 저장소로 변환 (.git 생성)
  3. git add .
  4. git status 로 확인
  5. git commit -m "메세지"
  6. git branch -M main // master 를 main 으로 이름 변경.
  7. git remote add origin http://원격저장소주소.com/username/repositoryName
  8. git push origin master

# git 기타 명령어

  • git remote -v : 현재 연결된 깃 주소를 보여줌.
  • git remote remove origin : 원격 저장소와 연결을 해제
  • git push origin --delete <브랜치이름> : 원격 저장소의 브랜치 삭제

# 7. 학습 내용에 대한 개인적인 총평

  • git 을 과제 제출용으로만 배워서, 지금껏 branch 라는 git 에서의 최고의 장점을 사용하지 않고 있었다. 코딩을 계속 하다보면 기존에 하던 작업을 나두고 여러개로 새롭게 작업해야 되는 경우가 생긴다. 이러한 코드를 지금까진 복사하고 새롭게 만들어서 붙여놓고, 이런식으로 진행하였는데, 브랜치를 만들면 작업하던 기록을 남길 수 있으며, 기존에 만든거에 영향을 미치지 않고 작업할 수 있다. 또한 작업 환경을 쉽게 이동할 수 있으며, 완료된 작업을 기존 작업과 병합하는 방법도 간단하다.
  • 차근차근 코딩 업무에 관한 정보들도 몸에 체득하여 실무에서 잘 할 수 있는 능력있는 인재로 성장하고 싶다. 노력하자.
