# 첫번째 방법

  • 파이널 시험때 구현했던 방법으로 다시 코드 구현함.
  • 함수 strtok 를 구현하여, c 를 '\0' 로 바꾼뒤, 각각의 주소값을 tab[i] 에 넣는 방식으로 만들어 봄.
  • 어디서 문제인지 세그먼트 폴트가 뜨는중.
  • 어디가 문제인지 찾지를 못하겠음. 문제 해결 필요.
  • 다른 부분 정리하면서 추후에 다시 만들어 보기로 결정함.

# 두번째 방법

  • substr 함수를 추가하여 잘린 부분을 담을 수 있도록 함수 설계.
  • 자른 갯수 + 1 만큼 malloc 을 tab 에 할당 해주고, tab[i] 마다 들어갈 내용만큼 malloc 을 할당해서 문자열을 복사한다.
  • 만들면서 메모리가 부족하여 malloc 이 할당되지 않는경우, free 함수를 통해 이전에 할당된 것들을 해제해켜 준다.

# <이차원 배열 적는 방법>

  • 자료형 배열이름[세로크기][가로크기];
  • 자료형 배열이름[세로크기][가로크기] = { { 값, 값, 값 }, {값, 값, 값} };
  • 문자 c 로 문자열 s를 나누어 각각의 문자열로 담는 함수의 모습.
    ex ) s = "a,b,cd" c = ","
	tab[0] = a
	tab[1] = b
	tab[2] = cd
	tab[3] = \0

# 7. 학습 내용에 대한 개인적인 총평

  • 코딩도장에 있는 strtok 함수와 비슷하게 파이널 시험 때 했었는데 그때 당시에는 출력에 대한 오류가 있었지만, 세그먼트 에러는 피했었다. 이번엔 그 부분에서 넘어가지 못한채 구현이 되지 않아서 기본적으로 strtok 함수가 어떻게 구현되어 있는지 다시 뜯어서 확인해볼 필요가 있을 것으로 생각이 된다.
  • 첫번째 방법의 핵심이 되는 기본적인 방법은 바꿀 c 값 대신 '\0' 을 넣어 자연스럽게 끊김과 동시에 각각의 맨 앞자리 주소값을 새로운 배열(포인터)에 담는 것이 원리이다.
  • 구현 아이디어는 좋지만 구현이 잘 되지않아서 문제점을 찾아야하지만, 오랫동안 잡고 있으면서 잘 풀리지 않을땐 다른 것들을 하면서 다시 해보는 것도 좋을 것 같다. 생각보다 libft 에서 시간을 많이 끌고 있기에 하루하루 더 학습에 신경을 쓰도록 노력해야겠다.
