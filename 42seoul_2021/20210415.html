# mlx_sync 사용법

mlx_sync 를 사용하면 화면 싱크를 맞출 수 있다.

맥북에서 계속 깜박이는 현상이 있었는데, mlx_loop_hook 이 너무 빨리 돌아 화면 싱크가 안맞기 때문에 발생하는 현상이었다. mlx_put_image_to_window 을 통해 이미지를 그려주는데, mlx_loop_hook 안에 이 함수가 있어서 이러한 문제가 발생한다.

# 사용법

image를 window에 그려주는 함수(mlx_put_image_to_window) 위, 아래에 sync함수를 넣자.

# 예시 코드
int draw(t_info *info)
{
	...
	mlx_sync(MLX_SYNC_IMAGE_WRITABLE, info->img.img);
	mlx_put_image_to_window(info->mlx_ptr, info->win_ptr, info->img.img, 0, 0);
	mlx_sync(MLX_SYNC_WIN_CMD_COMPLETED, info->win_ptr);
	return (0);
}
# 변경할 수 있는 부분

명령어는 둘 중 하나를 선택해서 사용하면 되고, MLX_SYNC_WIN_CMD_COMPLETED는 mlx_do_sync 함수로 대체하여 사용해도 기능은 같다.

// 2개중 택 1
mlx_sync(MLX_SYNC_IMAGE_WRITABLE, info->img.img);
mlx_sync(MLX_SYNC_WIN_FLUSH_CMD, info->win_ptr);

mlx_put_image_to_window(info->mlx_ptr, info->win_ptr, info->img.img, 0, 0);

// 2개중 택 1
mlx_sync(MLX_SYNC_WIN_CMD_COMPLETED, info->win_ptr);
mlx_do_sync(info->mlx_ptr);

# 함수 기본 구조

# mlx_sync 함수
#define MLX_SYNC_IMAGE_WRITABLE    1
#define MLX_SYNC_WIN_FLUSH_CMD     2
#define MLX_SYNC_WIN_CMD_COMPLETED 3

int	mlx_sync(int cmd, void *ptr);
// cmd : define 변수
// ptr : img_ptr 또는 win_ptr

MLX_SYNC_IMAGE_WRITABLE 는 이미지에 대한 모든 후속 호출을 버퍼링합니다(mlx img ptr 넣어야 함)

MLX_SYNC_WIN_FLUSH_CMD 는 이미지 변화를 감지해서 현재 이미지로 window를 새로고침한다.(win ptr)

MLX_SYNC_WIN_CMD_COMPLETED 은 처음 flush 이후, 다음 처리완료를 기다린다.

# mlx_do_sync 함수

기능은 모든 window에 cmd_completed 를 한 것과 같다.

int	mlx_do_sync(void *mlx_ptr);
// mlx_do_sync equals cmd_completed for all windows.

# 7. 학습 내용에 대한 개인적인 총평

  • 맵 파싱부터 다시 시작하기로 했다. 먼저 기존에 만들었던 구조를 다뜯어 고쳐야 해서 구조체 변수 수정과 makefile 수정, libft+gnl 추가 등의 작업을 진행하였다.
  • 맵 파싱이 단순히 값을 가져오기만 한다 생각을 했지만, 생각보다 오류처리해야 될 부분들이 많아서 시간적으로 오래 걸리는 작업이었다.
