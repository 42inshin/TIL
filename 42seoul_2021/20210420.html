# xpm 파일 이미지로 변환하기

각각의 img, data 가 있는 구조체를 만들어 mlx_xpm_file_to_image 함수를 이용하여 값을 불러 오자.

mlx_get_data_addr 함수로 받아온 tex.img 를 char * 형태로 데이터화 하자.

info->tex.n.img = mlx_xpm_file_to_image(info->mlx, info->tex.no, &info->tex.n.w, &info->tex.n.h);
if (!info->tex.n.img)
		return (error_check(info, 6));
info->tex.n.data = mlx_get_data_addr(info->tex.n.img, &info->tex.n.bpp, &info->tex.n.size_line, &info->tex.n.endian);

# 오류 체크

# 1. mlx_ptr, win_ptr 을 NULL 로 초기화 할 경우 문제 생긴다.

: mlxinit() 을 먼저 실행하고, 이후에 변수 초기화를 해줬기 때문에 생긴 문제이다. 변수 초기화 함수에 두 값을 제외 시키거나, 값 초기화를 제일 처음에 한다면 없어지는 문제!!

# 2. map_parser 에 memset에 0 을 넣을 때, 흰색으로 나오는 오류

왜 그런지 아직 못찾음.

현재는 공백(' ')으로 넣고, 색 칠하는 곳에 else 를 추가해서 0x00000000 을 넣음.

  • 해결 : strchr 함수를 사용하면 '\0' 까지 확인을 해서 0 값도 흰색에 포함이 되어버린다.

# 3. map.size가 특정 값 이하로 내려갈 경우, 캐릭터가 움직이지 못한다.

  • 해결 : map.size 값이 int 이기 때문에 pos.speed 1보다 작아질 때 속도가 0이 되면서 멈춘것이다. (double) 값으로 형변환 하거나 나눠주는 값을 16 대신 16.0 으로 하자.
  • map.size를 for 문에 넣어 이미지를 만들기 때문에 map.size 자체를 double 로는 바꾸지 말자.
info->pos.size = (double)info->map.size / 2;	// 캐릭터 크기
info->pos.speed = (double)info->map.size / 16; 	// 캐릭터 속도
// 또는
info->pos.size = info->map.size / 2.0;   // int / double 이라 double로 변환 된다.
info->pos.speed = info->map.size / 16.0;

# 4. planeX,Y 값을 회전 시키기
