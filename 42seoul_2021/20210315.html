# 비동기 처리 기법(callback, promise, async await)

자바 스크립트는 비동기 방식으로 코드를 읽어드린다. 반대말의 동기 방식은 요청-응답 후 값을 받고 다음 함수를 처리하는 것이다. 요청-응답 여부의 상관없이 빠르게 다음 코드를 실행하는 것을 비동기 방식라고 한다.

비동기 방식의 장점은 빠르게 페이지를 로드할 수 있고, 페이지 리로드 시 불필요하게 전체 코드를 다시 불러올 필요가 없어 리소스 낭비를 막아준다. 하지만 js를 통해 다른 곳에서 데이터를 얻어와 그 데이터를 처리를 해야할 경우에 문제가 된다. 아직 데이터를 얻기도 전에 다음 코드가 실행되는 문제가 발생하는 것이다.

그 문제를 해결하기 위해서 마치 동기 처리가 되는 구조를 만들어줘야하는데, 이걸 위해서 callback 함수, promise, async await 기법이 생겼다.

콜백 함수는 다른 코드의 인수로서 넘겨주는 “함수”이다. 부모 함수 내에 자식 함수가 부모 함수의 인자로 되어 있는 구조.. 그냥 찾아보자.. 아무튼 이 구조를 사용하면 함수 실행 시 자식함수가 만들어져야 부모함수가 끝나는 구조가 된다. 즉, 데이터를 다 읽은 후 함수가 처리되는 구조가 되는 것이다.

그러다가 객체 방식으로 더 쉽게 처리하기 나온게 Promise 이다. promise() 객체엔 데이터를 읽어드리는데 성공(이행 fufill), 실패(거부 reject)에 따라 결과 값을 사용할 수 있는 메서드가 있다. 그 메서드는 .resolve() .reject() 임. (메서드는 객체 안에 있는 함수를 부르는 말이다). 프로미스의 장점은 여러 개의 프로미스를 연결할 수 있다는 점! .then() 메서드를 이용하여 연결해준다.

async, await 는 비동기 처리 패턴 중 가장 최근에 나온 문법이다. (콜백 함수와 프로미스의 단점을 보완하기 위해 나온 문법!) 우리가 자주 쓰는 문법에서 async 와 await 만 붙이면 해결되어 구조가 짧고 가독성이 좋다. 위의 callback, promise 의 코드에 익숙하지 않아도 비동기 처리 코드를 만들 수 있다는 것이 장점!
