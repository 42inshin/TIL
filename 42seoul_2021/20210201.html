# kyukim - gnl 피드백

  1. static 을 사용 시 변수의 값 초기화.
    static char *str 로 선언하면 메모리 할당을 하지 않아도 0x0 으로 초기화된다.

    • 문제점
      => str = NULL일 때, *str 로 접근하는 코드 작성 시 세그폴트 에러.
    • 해결 방법
    if (str)
	while (*str)
	 	*tmp++ = *str++;

  2. main.c를 만드는 경우, char **line 을 만들어 gnl 함수에 사용하게 되면 세그폴트가 난다.

    • 이유
      이중포인터로 선언한 **line 변수가 받을 포인터의 메모리 주소가 없기 때문에 세그폴트가 난다.
    • 해결 방법
      char *line 을 선언하고 gnl엔 &line 인자로 넣으면 새그폴트가 나지 않는다. line 의 값(포인터의 값 = 주소값)을 gnl 안에서 바꿔주기 때문에 포인터의 주소값인 이중포인터 형태로 넣어주는 것이다.

# 7. 학습 내용에 대한 개인적인 총평

  • 동료학습을 하면서 기존에 만들었던 코드도 새로운 방법으로 만든 것을 보고 피드백하다보니 코딩력(?)이 확실히 올라가는 것을 느낀다.
  • 코드 하나를 같이 보면서 서로 피드백하면서 코드를 새롭게 짤 때마다 각자의 실력향상과 팀워크가 느는 점에 있어서 계속해서 동료학습을 잘 이어나갈 수 있길 기대해본다.
  • cub3d 의 경우, 전체적으로 어떤 것부터 해야되는지에 대해서 살펴 보았는데 아무래도 minilibX 가 어떤 것이고 어떻게 사용해야되는지 부터 알아야 될 듯 하다. 이 부분은 내일부터 다시 연구해봐야 되겠다.
