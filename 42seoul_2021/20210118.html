# ft_server

  1. 도커 설치

  2. 도커에 이미지 생성(debian:buster)
    : docker pull debian:buster

  3. 도커에 컨테이너 생성(debian:buster)
    : docker run -it —name con_debian -p80:80 -p433:433 debian:buster

  4. 컨테이너에 apt-get 최신버전으로 업데이트
    : apt-get updateapt-get upgrade

  5. 컨테이너에 nginx, cURL 설치
    : apt-get install -y nginxapt-get install -y curl

  6. 서버 nginx 실행 및 확인
    : service nginx startservice nginx status

  7. self-signed SSL 인증서 설치
    : apt-get install -y openssl

  8. SSL 인증서 생성
    : openssl req -newkey rsa:4096 -days 365 -nodes -x509 -subj "/C=KR/ST=Seoul/L=Seoul/O=42Seoul/OU=inshin/CN=localhost" -keyout localhost.dev.key -out localhost.dev.crt

  9. 권한 제한 및 파일 이동
    : chmod 600 localhost.dev.crt localhost.dev.keymv localhost.dev.crt etc/ssl/certs/mv localhost.dev.key etc/ssl/private/

  10. nginx 에 ssl 설정 및 url redirection 추가

    1. vim 설치
      : apt-get -y install vim
    2. 기본 파일 변경
      : vim etc/nginx/sites-available/default
    3. 변경 코드
    server {
	listen 80;
	listen [::]:80;

	return 301 https://$host$request_uri;
}

server {
	listen 443 ssl;
	listen [::]:443 ssl;

	# ssl 설정
	ssl on;
	ssl_certificate /etc/ssl/certs/localhost.dev.crt;
	ssl_certificate_key /etc/ssl/private/localhost.dev.key;

	# 서버의 root디렉토리 설정
	root /var/www/html;

	# 읽을 파일 목록
	index index.html index.htm index.nginx-debian.html;

	server_name ft_server;
	location / {
		try_files $uri $uri/ =404;
	}
}

  11. 서버 nginx 재시동 : service nginx reload

  • 여기까지 진행시 http -> https 로 리다이렉션 가능
  • macbook chrome 의 경우, 신뢰하지 못하는 사이트로 나올때 그 페이지에서 아래의 코드를 써주면 접속이 가능해진다.
    • thisisunsafe : 웹브라우저에서 이렇게 쓰면 접속 가능해짐.

# 7. 학습 내용에 대한 개인적인 총평

  • 처음에 server에 대한 지식이 전혀 없어서 어떤식으로 해야 될지 막막했으나 1기분들의 정리한 글들이 블로그에 다수 있어서 큰 도움이 되었다. ft_server의 경우엔 알아야 될 정보들이 많고, 각각의 내용들이 모두 생소하여 어렵지만 정리된 글들을 통해서 한땀한땀 알아가고, 실행해보면서 도커가 어떻게 사용되고 어떻게 명령어를 사용하는지, 그 안에서 server 구축이 되려면 어떻게 진행해야 되는지를 학습해볼 수 있었다.
  • 물론 알아야 하는 자세한 부분들이 많지만 학습을 진행하면서 그저 이해가 아닌 정리를 하면서 학습을 한다면 더 자세하게 ft_server 과제에 필요한 내용들을 파악할 수 있지 않을까 싶다.
  • 아직까진 Dokerfile 을 만드는 부분과 다른 src 만드는 것에 대해서 알아보진 못했으나 점차 익숙해진다면 쉽게 ft_server를 통과할 수 있지 않을까 생각해본다.
