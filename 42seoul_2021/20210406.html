# 6. 상세 학습 내용

실제 코딩 시간 : 총 3시간.

자바스크립트 코드를 head에 작성할 경우, html 요소들을 인지하기 전에 스크립트를 로드하게 된다.

이 경우, 페이지가 제대로 동작하지 않을 가능성이 높다.

자바스크립트에 다음과 같이 구성하면 해결할 수 있다.

# DOMContentLoaded 이용

이방법은 DOM content를 다 읽은 후에 스크립트 코드를 읽어 순서를 보장한다.

document.addEventListener("DOMContentLoaded", function() {
  ...
});

# <script></body> 작성

가장 평범한 방법으로 HTML, body 태그 마지막 위에 script를 작성하는 것이다.

	...
	<script src="script.js"></script>
</body>

# 위 두 방법의 문제점

위 두가지 방법은 HTML DOM 이 로드되기 전까지 script의 로딩과 파싱이 완전히 차단된다.

스크립트 코드가 많은 큰 규모의 사이트의 경우 사이트를 느리게 만드는 중요한 성능 문제를 야기할 수 있다.

# async 사용

최신 자바스크립트 문법인 **async** 를 html 에 사용할 수 있다.

기존에는 script 코드를 먼저 만나면 다 읽은 후 html 문서를 읽었지만, async 를 사용 시에 비동기적으로 html 요소를 멈추지 않고 읽어나간다.

<script **async** src="script.js"></script>

이 경우 script와 HTML 은 동시에 로드되고 작동할 것이다.

async 스크립트는 페이지 렌더링의 중단 없이 스크립트를 다운로드 하고, 또한 스크립트의 다운로드가 끝나자 마자 이를 실행시킵니다. async는 외부 스크립트끼리의 구체적인 실행 순서는 보장하지 않고, 단지 나머지 페이지가 나타나는 동안 스크립트가 비동기방식으로 다운로드 되어 중단되지 않는다는 것만 보장합니다. async는 각각의 스크립트가 독립적으로, 서로에게 의존하지 않는 관계일 때 적절합니다.

# defer

만약 script 를 읽는 순서가 중요할 경우에는 **defer** 가 더 적절하다.

<script **defer** src="js/vendor/jquery.js"></script>

<script **defer** src="js/script2.js"></script>

<script **defer** src="js/script3.js"></script>

위의 예제의 경우, jquery.js → script2.js → script3.js 순서가 보장된다.
