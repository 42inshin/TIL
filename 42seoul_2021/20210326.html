# 주요 사항

  • IR 기법을 사용하여 이미지로 배경 처리하고, 대체 택스트를 제공해야 한다.
  • HTML5 로 작성하며 의미에 맞는 태그를 최대한 사용한다.(시맨틱 마크업)
  • IE10 버전에도 호환될 수 있도록 코드를 구성한다.

# html5 핵심 요소 정리

Element Description
<header> HTML <header> 요소는 소개 및 탐색에 도움을 주는 콘텐츠를 나타냅니다. 제목, 로고, 검색 폼, 작성자 이름 등의 요소도 포함할 수 있습니다.
<footer> HTML <footer> 요소는 가장 가까운 구획 콘텐츠나 구획 루트의 푸터를 나타냅니다. 푸터는 일반적으로 구획의 작성자, 저작권 정보, 관련 문서 등의 내용을 담습니다.
<main> HTML <main> 요소는 문서 body의 주요 콘텐츠를 나타냅니다. 주요 콘텐츠 영역은 문서의 핵심 주제나 앱의 핵심 기능에 직접적으로 연결됐거나 확장하는 콘텐츠로 이루어집니다.
<nav> HTML <nav> 요소는 문서의 부분 중 현재 페이지 내, 또는 다른 페이지로의 링크를 보여주는 구획을 나타냅니다. 자주 쓰이는 예제는 메뉴, 목차, 색인입니다.
<section> HTML <section> 요소는 HTML 문서의 독립적인 구획을 나타내며, 더 적합한 의미를 가진 요소가 없을 때 사용합니다.
<aside> HTML <aside> 요소는 문서의 주요 내용과 간접적으로만 연관된 부분을 나타냅니다. 주로 사이드바 혹은 콜아웃 박스로 표현합니다.
<article> HTML <article> 요소는 문서, 페이지, 애플리케이션, 또는 사이트 안에서 독립적으로 구분해 배포하거나 재사용할 수 있는 구획을 나타냅니다.
<address> HTML <address> 요소는 가까운 HTML 요소의 사람, 단체, 조직 등에 대한 연락처 정보를 나타냅니다.
<strong> HTML <strong> 요소는 중대하거나 긴급한 콘텐츠를 나타냅니다. 보통 브라우저는 굵은 글씨로 표시합니다.

# css 주요사항

/* IR 기법을 위한 blind 처리 */
.blind {
	position: absolute;
	width: 1px; /* 0 이면 스크린 리더기에서 못 읽음 */
	height: 1px;
	margin: -1px;
	clip: rect(0 0 0 0);
	overflow: hidden;
}
/* Float 해제를 위한 코드 */
.clear::after {
	display: block;
	clear: both;
	content: '';
}
/* flex 자식요소 중앙 정렬 */
.parent_box {
	display: flex;
	justify-content: center;
	align-items: center;
}

# 7. 학습 내용에 대한 개인적인 총평

  • html, css를 계속 익숙해지기 위해서 네이버 부스트코스의 웹 UI 강의의 프로젝트 B를 만들어 보았다. 웹 UI의 경우, 42서울 과제를 하면서도 계속해서 기본 웹 지식을 놓치지 않기 위해서 올해 수강을 진행하여 정해진 시간 내에 끝마칠 수 있었다. 그 안에서 알고 있던 지식을 더 단단히 할 수 있었고, 웹 접근성에 대한 자세한 내용과 IR 기법 등의 새로운 지식을 습득할 수 있었다.
  • 이 과제를 통해서 간단하지만 IR기법을 사용할 때 코드를 어떻게 구성해야 할 것인지에 대해 더 심도 있게 고민해볼 수 있어서 많은 도움이 되었다.
  • 실제 업계로 나가서 나를 펼칠 수 있도록 지금의 학습에 더욱 정진하자.
