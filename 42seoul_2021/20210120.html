# ft_server

  • 어제 문제가 있던 부분은 wordpress 와 DB 연결이었다.

    mysql < var/www/html/phpmyadmin/sql/create_tables.sql -u root --skip-password

mysqladmin -u root -p password
기존 패스워드 없으니 엔터 입력

  • 위 코드에서 적은 아이디와 패스워드로 phpmyadmin 을 접속해보니 원활하게 잘 작동함.

# vim 팁 - ctags

  • jso 님 평가 때 알게된 vim 에서의 유용한 팁!

  • vim 에서
    :e. 명령어는 현재 디렉토리의 목록을 보여준다.

  • ctags 를 이용하면 파일 내부에서 함수를 따라 이동하며 볼 수 있다.
    mac에 ctags 설치 : brew install ctags

  • ctags *.c
    현재폴더에 tag file 생성

  • ctages -R
    전체 파일 및 하위 폴더의 파일도 tags 파일 만듦

  • vim 내부에선
    :!ctags *.c

  • :ta keyword
    keyword와 일치하는 태그가 선언되어 있는 곳으로 이동

  • :ta /keyword
    keyword를 포함하는 태그가 선언되어 있는 곳으로 이동

  • :tj keyword (jump)
    keyword와 일치하는 태그의 목록을 출력하고, 사용자가 선택하여 이동. (한 개일 경우 바로 이동)

  • :tn (next)
    목록의 다음 항목으로 이동 (tj 명령어를 사용하여 출력한 목록에서 5번 항목을 선택한 후 :tn 명령어를 입력하면 6번 항목으로 이동합니다.)

  • :tp (previous)
    목록의 이전 항목으로 이동 (tj 명령어를 사용하여 출력한 목록에서 5번 항목을 선택한 후 :tp 명령어를 입력하면 4번 항목으로 이동합니다.

  • :tr (rewind) / :tf (first)
    목록의 제일 처음 항목으로 이동

  • :tl (last)
    목록의 제일 마지막 항목으로 이동

  • :tjs keyword (jump)
    tj 명령어를 새창에서 실행합니다.

소스 코드 분석 중 현재 커서의 위치에 있는 태그의 선언 부분을 보고 싶으시면 명령어 입력 없이

  • Ctrl + ]

만약 이전 위치로 돌아가고 싶다면

  • Ctrl + t

# 7. 학습 내용에 대한 개인적인 총평

  • 평가를 위해서 받았던 포인트를 기부하고 다시 평가를 진행하였다. 확실히 평가를 진행하게 되면 여러 사람들을 만날 수 있고 그 안에서 배울 수 있는 내용이 많이 있다. 이번에 만나게 된 jso 님 덕분에 평가하는 방법에 대해서 생각해 볼 수 있는 시간이 있었으며, 유용한 팁들을 알려주면서 뭔가를 알려주기 보다는 더 알아가게 된 좋은 시간이었다.
  • ft_server 에서의 문제점을 계속 찾기 어려웠지만, 지나간 카뎃 분들의 정리 덕분에 수월하게 과제에 대한 내용을 습득할 수 있었다. 어제 문제가 되던 부분을 수정하니 wordpress 부분까지 해결이 되었고, 이제 dockerfile을 만드는 법에 대해서 익히면 될 듯하다.
  • 아직까지 정확한 개념은 다 이해하지 못했기 때문에 더 부지런하게 정리를 해 server 에서 필요한 개념들을 확실히 익히고 넘어가도록 하자.
