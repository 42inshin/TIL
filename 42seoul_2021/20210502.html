# make 시 mlx.dylib 경로 문제 추가 내용

동적라이브러리의 다른 말은 실행 경로 의존 라이브러리

해당 실행 파일의 동적라이브러리 경로를 확인 할 수 있는 방법

: otool -L 실행파일

otool -L cub3D

cub3D:
	libmlx.dylib (compatibility version 0.0.0, current version 0.0.0)
	/System/Library/Frameworks/OpenGL.framework/Versions/A/OpenGL (compatibility version 1.0.0, current version 1.0.0)
	/System/Library/Frameworks/AppKit.framework/Versions/C/AppKit (compatibility version 45.0.0, current version 2022.20.117)
	/usr/lib/libSystem.B.dylib (compatibility version 1.0.0, current version 1292.60.1)

위 예시를 보면, libmlx.dylib 경로는 현재 실행 파일 위치를 가르킨다.

같은 위치에 libmlx.dylib 파일이 없기 때문에 오류가 발생힌다.

아래의 명령어로 실행파일의 동적라이브러리 경로를 변경할 수 있다.

install_name_tool -change libmlx.dylib ./mlx/libmlx.dylib cub3D

이 코드를 makefile 에 추가하여 올바른 경로로 수정해주면 mlx 파일을 같은 경로에 두지 않아도 잘 읽어서 실행파일이 실행 된다.

참고 링크 : https://jacobpan3g.github.io/2017/09/01/build-dylib-on-mac/ (opens new window)

# 동적 라이브러리 vs 정적 라이브러리

  • 동적 라이브러리 : 완성된 프로그램을 실행할 때 포함시키는 라이브러리

  • 프로그램 변경시 변경된 부분의 공유 라이브러리만 재배포하면 되므로 유지보수가 쉽다.

  • 외부 의존도가 생기기 때문에 이식성이 어렵다.

  • 정적 라이브러리 : 프로그램을 컴파일하는 과정에서 포함시키는 오브젝트 파일들의 모임

  • 컴파일의 링킹 단계에서 실행 파일에 결합 된다. 실행 바이너리에 포함시킨다는 의미

  • 실행 파일에 다 들어가기 때문에 실행 시 라이브러리가 필요 없다.

    참고링크 : https://goodgid.github.io/Static-VS-Dynamic-Libray/ (opens new window)
