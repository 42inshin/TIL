# lodev 아이디어

  1. 벽을 레이 캐스팅하는 동안 각 수직 스프라이트의 수직 거리를 1D ZBuffer 에 저장한다.
  2. 각 스프라이트에서 플레이어까지의 거리를 계산한다.
  3. 이 거리를 사용하여 카메라에서 가장 먼 곳에서 가장 가까운 곳까지의 스프라이트를 정렬한다. (먼 곳부터 가까운 순으로 그리자)
  4. 카메라 평면(plane in 2D)에 스프라이트를 투영하자 : 스프라이트 위치에서 플레이어 위치를 뺀 다음, 그 값에 2x2카메라 매트릭스(행렬)의 역수를 곱한다.
  5. 수직 거리를 사용하여 화면 (x, y direction 모두)에서 스프라이트의 크기를 계산한다.
  6. 스프라이트 수직 선에 따라 수직 선으로 그려라. 거리가 1D ZBuffer보다 멀면 수직선을 그리지마라.

링크 : https://lodev.org/cgtutor/raycasting3.html (opens new window)

# 해야할 것

  • map 에 2가 들어올 경우 스프라이트로 인식
  • 맵에 있는 스프라이트 개수를 세기
  • 각 스프라이트의 위치 값 보정 (+0.5)
  • 현재 위치와 스프라이트 위치에 대한 거리를 구하자.
  • 각 스프라이트를 현재 위치에서 먼 순서부터 가까운 순서로 정렬(sort_sprites)하자.

# sprite dist 내림차순 정렬

멀리 있는 스프라이트부터 출력해야 하므로 내림차순으로 정렬 하자.

// 선택정렬
void	sort_sprite(t_info *info)
{
	t_sprite	tmp;
	int			i;
	int			j;

	i = 0;
	while (i < info->map.sprite_num - 1)
	{
		j = i + 1;
		while (j < info->map.sprite_num)
		{
			if (info->spr[i].dist < info->spr[j].dist)
			{
				tmp = info->spr[i];
				info->spr[i] = info->spr[j];
				info->spr[j] = tmp;
			}
			j++;
		}
		i++;
	}
}

출력했을 시, 정렬이 잘 된 것을 확인 할 수 있었다.

# 7. 학습 내용에 대한 개인적인 총평

  • sprite 의 경우, 지금까지 했던 것들과 비교했을 때 가장 이해가 가지 않는 부분인거 같다. 며칠째 계속 글을 읽으면서 이해해보려고 했으나, 카메라 매트릭스의 역수를 왜 곱해야하는지에 대한 궁금증이 가시질 않아서 좀처럼 진도가 나가지 않는 상황이다. 일단 크기에 상관없이 그 위치에 스프라이트를 출력하고, 크기와 방향을 보정하는 방식으로 진행해보도록 해야겠다.
