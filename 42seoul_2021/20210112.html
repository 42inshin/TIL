# CSS

: Cascading Style Sheets

  • 연속적으로 계속되는 스타일 규칙

h1 {color: yellow; font-size:2em;}

  • 선택자(selector) - "h1"
  • 속성(property) - "color"
  • 값(value) - "yellow"
  • 선언(declaration) - "color: yellow", "font-size: 2em"
  • 선언부(declaration block) - "{ color: yellow; font-size:2em; }"
  • 규칙(rule set) - "h1 { color: yellow; font-size:2em; }"

부분 속성값

  • [class~="bar"] : class 속성의 값이 공백으로 구분한 "bar" 단어가 포함되는 요소 선택
  • [class^="bar"] : class 속성의 값이 "bar"로 시작하는 요소 선택
  • [class$="bar"] : class 속성의 값이 "bar"로 끝나는 요소 선택
  • [class*="bar"] : class 속성의 값이 "bar" 문자가 포함되는 요소 선택

문서 구조 선택자

  • 자손 선택자 : div span
  • 자식 선택자 : div > span
  • 인접 형제 선택자 : div + ul div 바로 옆에 있는 ul 선택
  • 인접 형재 선택자2 : div~p div 와 같은 형제인 p 를 선택

가상 클래스
:hover, focus, active, first-child, nth-child() 처럼 미리 정의해 놓은 상황에서 적용되도록 하는 “보이지 않는 클래스”

  • :link - 하이퍼링크이면서 아직 방문하지 않은 앵커
  • :visited - 이미 방문한 하이퍼링크인 앵커
  • :focus - tap 으로 입력된 상태(입력초점)
  • :hover - 마우스 올려진 상태
  • :active - 마우스 클릭된 상태

가상요소

  • :before - 가장 앞에 요소 삽입
  • :after - 가장 뒤에 요소 삽입
  • first-line - 요소의 첫 번째 줄에 있는 텍스트(browser의 첫줄)
  • :first-letter - 블록 레벨 요소의 첫 번째 문자(첫번째 글자)

구체성
:요소 선택의 규칙 1,0,0,0 인라인 0,1,0,0 id 0,0,1,0 클래스 0,0,0,1 모든 요소, 가상 요소 !important - 구체성 상관없이 우선권을 갖음 p#page { color: red !important; }

상속
: css 상속은 박스모델은 상속되지않음 구체성도 가지지 않아 구체성 상관없이 * 로 지정한게 상속됨.

# docker 설치

  • 도커 홈페이지에서 로컬용 mac-install docker.dmg 다운로드
  • GUI 에 맞게 클릭 시 terminal 을 통해 docker가 설치 진행 됨

clone
The Getting Started project is a simple GitHub repository which contains everything you need to build an image and run it as a container. Clone the repository by running Git in a container.

$docker run --name repo alpine/git clone https://github.com/docker/getting-started.git
$docker cp repo:/git/getting-started/ .

You can also type the command directly in a command line interface.

Build
A Docker image is a private file system just for your container. It provides all the files and code your container needs.

$cd getting-started
$docker build -t docker101tutorial .

Run your first container
Start a container based on the image you built in the previous step. Running a container launches your application with private resources, securely isolated from the rest of your machine.

getting-started$ docker run -d -p 80:80 --name docker-tutorial docker101tutorial

Now save and share your image
Save and share your image on Docker Hub to enable other users to easily download and run the image on any destination machine.

$docker tag docker101tutorial inshin/docker101tutorial
$docker push inshin/docker101tutorial

# 7. 학습 내용에 대한 개인적인 총평

  • docker 설치와 도커가 어디에 사용되는지에 대해서 자료조사를 실시하였다. 아직까진 생소한 부분이여서 이해가 되지 않지만, 계속해서 내용파악을 하면서 도커가 어떤 것인지 정리를 해둬야 하겠다.
  • 웹 UI 강의를 몰아서 듣고 있는 중이다. 알고 있던 내용들이 많지만 들으면서 처음보는 부분들도 있어서 다시 웹에 대해서 다시 정리가 되는 강의라 끝까지 시간내에 완주를 목표로 삼고 강의를 수강해야겠다.
  • 코딩을 하던 printf 와 달리 이해가 필요한 server 과제를 하다보니 스터디가 조금은 느슨해지는 것 같다. 어떤 방법으로 다시 스터디를 활성화 시킬지 고민이든다.
  • 사람과 사람 간의 소통과 기분을 어떻게 맞춰주면 좋을지 생각을 많이하는 요즘이다.
  • 그래도 꾸준히 목표를 향해 나아가고 있으니 분명히 좋은 일들이 있지 않을까 내심 기대해 본다.
